# 実PDF変換品質ログ

実際のPDF（合成PDFではない）でconvert.pyを動かし、変換品質を記録する。
目標は5〜10本でまず様子を見て、将来的に100本規模のベンチマークに広げること。

**匿名化について**: 検証には実務で使われている書類（請求書・見積書・契約書等）を
使用したが、本ログでは取引先名・現場名・個人名・具体的なファイルパスは記載せず、
サンプルID（S01〜）とカテゴリ・技術的な所見のみを記録する。

## 判定基準

- `○` 完全変換: 二重写りなし・全文字編集可能・レイアウト一致・警告なし（想定内の警告除く）
- `△` 一部制限あり: 変換は成立し崩れは無いが、Phase 1の設計上の理由
  （ページ回転・スキャン検出等）で一部/全部が編集不可のまま背景画像になる。
  レイアウト自体の破綻ではない
- `×` 実用不可: 二重写り・欠損・レイアウト破綻・クラッシュ等、実用に耐えない

各PDFについて次の9点を確認する: ①変換成否 ②warning内容 ③redaction後背景の
二重写り有無 ④画像/印影/罫線の欠損有無 ⑤PPTX内の編集可能テキスト数
⑥ページごとのフォールバック有無 ⑦元PDFとの近似レンダリング比較
⑧判定 ⑨崩れ理由・次フェーズ改善候補

**まとめ**: 10件中 ○7件・△3件・×0件。二重写り（redaction漏れの自己検査警告）は
**全件で0件**。×判定は無く、△はすべて「Phase 1の設計として意図的に編集不可へ
倒しているケース」（ページ回転・スキャン検出）であり、レイアウト破綻や欠損は
一度も発生していない。表が非常に密なケース（S04: 388テキストボックス、S05: 138
テキストボックス）でも罫線・チェックボックス・印影の欠損は無かった。

## 崩れ分類（△・×が出たら記録）

現時点で「実際の変換崩れ」に分類される事例は0件。△判定はすべて次の2パターンで、
どちらもPhase 1の安全側フォールバック（README「Phase 1の範囲」参照）が正しく
作動した結果であり、バグではない。

| パターン | 該当 | 次フェーズでの改善余地 |
|---|---|---|
| ページ回転（/Rotate） | S07, S08 | 回転ページの座標補正込み編集対応（README「Phase 2以降の候補」に既記載） |
| スキャンPDF（テキスト層なし） | S09 | ローカルOCRの適用（README「Phase 2以降の候補」に既記載） |

一度だけ**誤検知（レビュー時の見間違い）**があったので記録しておく: S02の
redaction後背景でタイトル部分が黒い塗り矩形だけになっており一見「文字が消えて
戻っていない＝欠損」に見えたが、実際はタイトルが白抜き文字（黒帯に白文字）で、
白文字だけが正しくredactionで消え、黒帯（図形）は保持されていた。PPTX側で該当
テキストボックスがfont.color=FFFFFFで存在することを確認し、シミュレーション
画像でも黒帯の上に白文字が正しく再現されることを確認済み。**背景PNG単体だけを
見て「文字が消えている」と判断するのは誤りで、必ずPPTX側のテキストボックスの
存在・色まで確認する**というレビュー手順上の教訓として残す。
